# v0.1.0 — Private Prototype Integrity Release

**Release date:** 2026-05-03
**Status:** Private academic prototype — not for public distribution.

---

## What is included

- **Curated qira'at explorer UI** — React/Vite application displaying textual variants across canonical recitation traditions (Hafs, Warsh, Qalun, and others) with tooltips, legends, and source attribution.
- **Canonical `quranData.json`** — 8 curated verses, 8 variant groups, 53 readings. All Arabic text is hand-sourced from Tanzil, KFGQPC, or classical qira'at literature. `_meta.generated = false`.
- **Verification ledger** — `verificationLedger.json` separates the review workflow from the dataset. Each reading has a `VerificationRecord` for tracking reviewer sign-off.
- **Audit system** — `npm run audit:data` cross-checks dataset/ledger integrity and exits non-zero on hard errors.
- **Provenance tracking** — `npm run record:provenance` hashes local corpora and records metadata without copying Arabic text into reports.
- **Source matching** — `npm run match:sources` performs diagnostic character-for-character comparison against local corpora. Replacement eligibility is gated by source authority level.
- **Source authority safeguards** — `replacementEligibility` requires `official_confirmed` or `scholar_confirmed` provenance. Unknown-provenance corpora produce diagnostic matches only, never source ID replacements.
- **HarfBuzz optional fallback** — App degrades gracefully to browser-native shaping when the WASM blob or Qur'anic font is absent.
- **CI workflow** — GitHub Actions runs audit, tests, and build on push/PR to `main`.
- **122 passing tests** across 10 test files covering integrity, provenance, source matching, Arabic normalisation, alignment, components, and audit rules.
- **ADRs** — Four architectural decision records under `docs/decisions/`.
- **Private contribution policy** — `CONTRIBUTING.md` restricts contributions to the core research team until publication.

## What is not included

- Full Qur'an coverage (only 8 curated verses).
- Official confirmed corpora (all local sources have `authorityStatus: unknown`).
- Any `confidence: verified` readings (none exist; all groups are `curated_sample`).
- AI-generated Arabic text (none exists; policy strictly forbids it).
- Raw corpus files (`.gitignore` excludes `scripts/_raw/`).
- HarfBuzz WASM or bundled Qur'anic fonts.

## Sacred text integrity guarantees

- `_meta.generated: false`
- `_meta.generationPolicy: "no-arabic-text-generation"`
- All Arabic text deep-frozen at runtime; mutation throws in strict mode.
- `manual_placeholder` cannot coexist with `confidence: verified`.
- Source matching never generates, modifies, or synthesises Arabic text.
- No automated promotion to `verified` — requires human reviewer signature in the ledger.

## Known limitations

- **manual_placeholder count: 12** — Twelve Warsh/Qalun readings have `manual_placeholder` as their source because the local corpora lack confirmed provenance.
- **Source authority: all unknown** — All three local corpora (`tanzil-hafs-uthmani`, `kfgqpc-warsh`, `kfgqpc-qalun`) have `authorityStatus: unknown`. No source has `official_confirmed` or `scholar_confirmed`.
- **Duplicate corpus hashes** — The Warsh and Qalun local files share an identical SHA-256 hash, confirming they are subset test fixtures, not full official distributions.
- **No scholar verification** — Zero readings have been independently verified by a qira'at scholar. All 53 readings are `unreviewed` in the ledger.
